How I Did

The Technology

From learning enums to debugging IR sensors — here is the tech that made the robot come alive.

🧠 Concept

Enumeration (Enums)

A new coding concept I learned during the competition. Enums let me create custom data types for robot states — making the code far cleaner than raw integers and enabling robust state machines.

🔌 Electronics

Potential Divider Circuits

Understood how voltage dividers work with LDRs and IR sensors. This was critical for tuning sensor thresholds so the robot could reliably distinguish black lines from the floor and detect tunnel darkness.

🤖 Robot

Line-Following Robot

Built with an Arduino Uno, 2 IR sensors for line tracking, an HC-SR04 ultrasonic sensor for obstacle detection, an LDR for tunnel detection, and an L293D motor driver.

💡 Task

Automatic Headlights in Tunnels

Used an LDR (light-dependent resistor) on A0 to sense ambient light. When the robot entered a dark tunnel (analogRead < 450), an LED automatically turned on — just like real car headlights!

🚧 Task

Obstacle Avoidance

Implemented multiple strategies — from simple delay-based sequences to a parallel-pass state machine that turns alongside the obstacle, passes it, and actively re-acquires the line using a sweep scan with confirmation debounce.

🔄 Task

U-Turn & Return

When both IR sensors detected the line simultaneously (end of map), the robot performed a 180° U-turn and followed the line all the way back to the start — demonstrating bidirectional navigation.
